# memoryfishing
# 🎣 LED 기억력 낚시 게임 (Raspberry Pi Pico 2 W)

제작자: 이해준(당곡고 10120), 정해성(신림고 , 20223), 오윤서(수도여고, 10915)
라즈베리파이 피코 2 W와 WS2813 LED 바를 이용한 **기억력 게임**입니다.
LED 바에 순서대로 켜지는 물고기(진짜 1마리 + 가짜 7마리) 중에서
**진짜 물고기(빨간색)의 위치를 기억**했다가, 스마트폰 웹 대시보드에서
정답 칸 번호를 입력하는 게임입니다.

## 🎮 게임 소개

1. LED 바(60개)에 물고기들이 **2개씩 묶여서 순서대로** 깜빡입니다.
   - **빨간색** = 진짜 물고기 (1마리, 정답)
   - **주황색** = 가짜 물고기 (7마리, 미끼)
2. 모든 물고기가 한 번씩 보여지고 나면, LED 바는 **보라색**으로 바뀌고
   플레이어가 정답을 입력할 차례가 됩니다.
3. 스마트폰(웹 브라우저)에서 진짜 물고기가 있던 **칸 번호(0~59)**를 입력합니다.
4. 정답이면 LED가 **초록색**으로 반짝이고, 오답이면 정답 위치가 **빨간색**,
   내가 입력한 위치가 **파란색**으로 표시됩니다.
5. 결과 확인 후 자동으로 다음 라운드가 시작됩니다.

> 💡 물고기의 실제 위치는 오직 **LED 바**에서만 확인할 수 있으며,
> 웹 대시보드에는 위치 정보가 노출되지 않습니다. (순수하게 기억력으로 승부!)

## 🛠 필요한 준비물

| 부품 | 사양 |
|------|------|
| 보드 | Raspberry Pi Pico 2 W |
| LED | WS2813 LED 스트립 (60구) |
| 전원 | 5V 외부 전원 (LED 개수에 맞는 충분한 전류 공급 필요) |
| 기타 | 점퍼 케이블, 브레드보드 등 |

## 🔌 회로 연결

| Pico 2 W | WS2813 |
|----------|--------|
| GPIO 18  | DIN (데이터 입력) |
| GND      | GND (공통 접지 필수) |
| 5V (외부 전원) | VCC |

> ⚠️ LED 60개를 모두 켤 경우 소비 전류가 크므로,
> **Pico 보드 자체 3.3V가 아닌 별도의 5V 전원**을 사용하세요.
> 이때 Pico와 LED 전원의 **GND는 반드시 공통으로 연결**해야 합니다.

## 💾 설치 방법

1. [MicroPython (Raspberry Pi Pico 2 W용)](https://micropython.org/download/RPI_PICO2_W/)을
   피코에 설치합니다.
2. 아래 두 파일을 피코에 업로드합니다. (Thonny 등의 IDE 사용)
   - `main.py`
   - `wifi_config.py` (아래 형식으로 직접 생성)

```python
# wifi_config.py
WIFI_SSID = "본인의 와이파이 이름"
WIFI_PASSWORD = "본인의 와이파이 비밀번호"
```

3. 피코를 재부팅하면 자동으로 `main.py`가 실행됩니다.
4. 시리얼 콘솔(Thonny 등)에서 출력되는 **IP 주소**를 확인합니다.

```
와이파이 연결 중.....
연결 성공! IP 주소: 192.168.0.XX
웹 서버 시작됨! 스마트폰에서 아래 주소로 접속하세요:
http://192.168.0.XX/
```

5. 같은 와이파이에 연결된 스마트폰/PC 브라우저에서 해당 주소로 접속합니다.

## 🕹 게임 방법

1. 웹페이지에서 **[게임 시작]** 버튼을 누릅니다.
2. LED 바를 잘 보고, **빨간색으로 켜지는 진짜 물고기의 위치(칸 번호)**를 기억합니다.
   - LED는 왼쪽부터 0번, 오른쪽 끝이 59번입니다.
3. 모든 물고기가 다 보여지면 입력창이 활성화됩니다.
4. 정답이라고 생각하는 칸 번호(0~59)를 입력하고 **[제출]**을 누릅니다.
5. 결과를 확인하고, 자동으로 시작되는 다음 라운드에 도전합니다!

## 🗂 코드 구조

```
.
├── main.py           # 메인 게임 로직 + 웹 서버
└── wifi_config.py    # 와이파이 접속 정보 (직접 생성 필요)
```

### `main.py` 구성

| 섹션 | 설명 |
|------|------|
| WS2813 LED 제어 | PIO(rp2)를 이용한 저수준 LED 데이터 전송 |
| 게임 상태(`GameState`) | 라운드 진행, 물고기 위치, 점수 관리 |
| 게임 로직(`game_step`) | showing → guessing → result 단계 자동 전환 |
| LED 렌더링(`render`) | 현재 게임 단계에 따라 LED 색상 결정 |
| 와이파이 연결 | STA 모드로 공유기에 접속 |
| 웹 대시보드 (HTML) | 점수/상태 표시, 정답 입력 UI |
| 웹 서버 | `/status`, `/answer`, `/start` API 제공 |
| 메인 루프 | 요청 처리 + 게임 진행 + LED 렌더링 반복 |

### 게임 단계 (Phase)

```
WAITING (대기) → SHOWING (물고기 순차 점등) → GUESSING (정답 입력 대기) → RESULT (결과 표시) → (다음 라운드로 자동 반복)
```

### API 엔드포인트

| 경로 | 설명 |
|------|------|
| `GET /` | 웹 대시보드 HTML 반환 |
| `GET /status` | 현재 게임 상태(JSON) 반환 |
| `GET /start` | 새 게임 시작 (점수/라운드 초기화) |
| `GET /answer?value=N` | N번 칸을 정답으로 제출 |

## ⚙️ 주요 설정값 (main.py 상단)

| 변수 | 기본값 | 설명 |
|------|--------|------|
| `NUM_LEDS` | 60 | LED 개수 |
| `MAX_BRIGHTNESS` | 60 | 최대 밝기 (0~255) |
| `LIGHT_ON_MS` | 300 | 물고기 묶음이 켜져 있는 시간 (ms) |
| `LIGHT_OFF_MS` | 150 | 다음 묶음 전 꺼져 있는 시간 (ms) |
| `GROUP_SIZE` | 2 | 한 번에 동시에 켜지는 물고기 수 |
| `NUM_FAKE` | 7 | 가짜 물고기 개수 (진짜 1 + 가짜 7 = 총 8마리) |

> 게임 난이도를 조절하고 싶다면 위 값들을 수정해보세요.
> 예) `NUM_FAKE`를 늘리면 더 어려워지고, `LIGHT_ON_MS`를 줄이면 더 빠르게 진행됩니다.

## 🔧 문제 해결

| 증상 | 확인해볼 사항 |
|------|----------------|
| 와이파이 연결 실패 | `wifi_config.py`의 SSID/비밀번호 확인, 2.4GHz 와이파이인지 확인 |
| LED가 켜지지 않음 | GPIO 핀 번호(18), 배선, 외부 전원 공급 여부 확인 |
| LED 색이 이상함 | WS2813은 GRB 순서 사용 (코드에서 이미 처리됨) — 다른 LED라면 순서 확인 필요 |
| 웹페이지 접속 안 됨 | 스마트폰과 피코가 **같은 와이파이망**에 있는지 확인 |
| 게임이 느리거나 끊김 | `RENDER_INTERVAL`, `LIGHT_ON_MS` 등 타이밍 값 조정 |
